Change History

Version: 2.0; Status: Final

  • Queue Collection
    • Removal of the queue collection page
  • ITK Behaviour
    • This has replaced the old 'Services' section and now provide a more generic definition of ITK behaviour.
  • Acknowledgement Framework
    • Linked out as a separate DMS and can be accessed from the Development Resource section.
  • Development Resources
    • New page added to include technical files and documentation required to implement ITK.
  • Metadata
    • Linked out as a separate DMS and can be accessed from the Development Resource section.

Version: 1.1; Status: Final

  • Rebranding to HSCIC
    • Pack has been rebranded to HSCIC
  • Navigation
    • Removal of the Related Documents tab, as these documents now delivered in their own TRUD pack
    • Removed WSDL option under Services as links to WSDL files now directly from Services Overview page and Queue Collection page
    • Some items may appear under different tabs
  • Metadata
    • Addition of Metadata tab that includes metadata schema and examples
  • Acknowledgement Framework
    • Acknowledgement Framework talks about the addition of 2 new handling specifications for requesting an infrastructure acknowledgement and a business response
  • Distribution Envelope Examples
    • Examples updated to include the new Acknowledgement request flags

Version: 1.0; Status: Final

  • Related Documents
    • Changes to the Architecture Specifications (relaxing of criteria for accreditation). A summary of the changes can be found on TRUD under "ITK Issued Specifications", downloading the "NHS Interoperability Toolkit Security Requirements Accreditation and Concessions" pack.
  • Business Acknowledgement
    • Correction made to the business acknowledgement. Details of the changes can be found in the change history of The Domain Message Specification for the ITK Business Acknowledgement

Version: 1.0; Status: RC2

  • Infrastructure Acknowledgement Interaction
    • Corrected broken links to Distribution Envelope and WSDL
  • Distribution Envelope
    • Corrected examples to used correct service names, profile ids and interaction ids mimeType for CDA set to text/xml
  • metadata.xsd
    • Introduced a new metadata.xsd schema that includes the rim.xsd schema. This mandates a <itk:metadata> tag around the metadata payload to ensure that the metadata payload is deterministic when it contains multiple extrinsic objects. All metadata payloads should be validated against metadata.xsd
  • Related Documents
    • Architecture Specification and Trust Operating Model documents updated. For details see the Change Log under Related Documents
  • ProfileId
    • ProfileId expanded to encompass Message Type, Template Constraints and Core Schema
